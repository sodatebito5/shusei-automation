# 要件定義 - 守成クラブ福岡飯塚ダッシュボード

## 🎯 プロジェクト目的

月例会運営における手作業・属人化を排除し、誰でも1時間で引き継げるシステムを構築する。

**設計思想**: 判断は人、処理はシステム

---

## 👥 ユーザー

| ユーザー | 用途 |
|----------|------|
| 会員 | 出欠連絡、売上報告、ゲスト申請（LIFF経由） |
| 運営担当 | ダッシュボードで全体管理、座席配置、式次第作成 |

---

## 📱 機能要件

### 1. 出欠連絡（LIFF）

**概要**: 会員が月例会への出欠を登録

| 項目 | 内容 |
|------|------|
| 入力 | 出席/欠席、備考 |
| 出力 | Google Sheets に記録 |
| 認証 | LINE ログイン（LIFF） |

**フロー**:
1. LINEからLIFFアプリ起動
2. LINE IDで本人特定
3. 出欠選択・送信
4. Sheetsに記録

---

### 2. 売上報告（LIFF）

**概要**: メンバー間の売上を報告

| 項目 | 内容 |
|------|------|
| 入力 | 相手先、金額、案件内容 |
| 出力 | Google Sheets に記録 |
| 認証 | LINE ログイン（LIFF） |

---

### 3. ゲスト申請（LIFF）

**概要**: 会員がゲストを招待する際の申請

| 項目 | 内容 |
|------|------|
| 入力 | ゲスト名、業種、連絡先 |
| 出力 | Google Sheets に記録 |
| 認証 | LINE ログイン（LIFF） |

---

### 4. 自動配席

**概要**: 出席者情報をもとに座席表を自動生成

| 項目 | 内容 |
|------|------|
| 入力 | 出席者リスト、テーブル配置 |
| 出力 | 座席表（画面表示 / PDF） |
| ロジック | 同業種分散、役職考慮 など |

---

### 5. ダッシュボード

**概要**: 運営担当が全体を管理する画面

**含む機能**:
- 出欠状況一覧
- 売上報告一覧
- ゲスト申請一覧
- 式次第プレビュー・編集・PDF出力
- 座席配置画面へのリンク

---

### 6. 式次第

**概要**: 月例会の進行表を作成・出力

| 項目 | 内容 |
|------|------|
| 入力 | 日付、会場、出席者情報（Sheetsから自動取得） |
| 出力 | PDF（印刷用） |
| 機能 | プレビュー、項目並び替え、編集 |

---

### 7. スケジュール管理

**概要**: 月例会の日程・会場を管理

| 項目 | 内容 |
|------|------|
| 入力 | 日付、会場、時間 |
| 出力 | 各機能で参照 |

---

## 🔗 データフロー

```
[会員] → LIFF → [GAS API] → [Google Sheets]
                                    ↓
[運営担当] ← ダッシュボード ← [GAS API]
                                    ↓
                              [PDF / 座席表]
```

---

## ⚙️ 非機能要件

| 項目 | 要件 |
|------|------|
| 可用性 | Google Workspace稼働時間に依存 |
| セキュリティ | LINE認証、GAS実行権限で制御 |
| 引き継ぎ | 1時間で運用開始できるドキュメント |
| 保守性 | コード・シート構造の明文化 |

---

## 📋 シート構成（想定）

| シート名 | 用途 |
|----------|------|
| メンバー一覧 | 会員マスタ（LINE ID紐付け） |
| 出欠管理 | 月例会ごとの出欠記録 |
| 売上報告 | 売上報告履歴 |
| ゲスト申請 | ゲスト申請履歴 |
| 会場マスタ | 会場情報 |
| スケジュール | 月例会日程 |

---

## 🚫 スコープ外（今回やらないこと）

- 会員登録機能（マスタは手動管理）
- 決済機能
- 通知機能（LINEへのPush通知）

---
